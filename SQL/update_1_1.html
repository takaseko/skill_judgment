1 更新関係に使うSQL SQLのデータ更新関係の命令は主に以下の3種類 1_1 UPDATE ・既存データの更新 ・新規作成や削除はできない点に注意 1_2 INSERT ・データの新規作成 ・データの更新はできない ・ユーザーを追加する時などに使用する 1_3 DELETE ・データの削除 ・削除なので特に気をつけないといけない命令 1_4 更新処理系に使うSQL文のまとめ ・新規データの追加・・・INSERT ・既存データの更新・・・UPDATE ・既存データの削除・・・DELETE 2 例で使用するテーブル （例）userテーブル ・idと名前とポイントが記録されているuserテーブル +------+--------+-------+ | id | name | point | +------+--------+-------+ | 1 | 山田 | 80 | | 2 | 鈴木 | 90 | | 3 | 加藤 | 60 | | 4 | 田中 | 70 | +------+--------+-------+ 3 「UPDATE」の使い方 3_1 基本的な書き方 UPDATE [テーブル名] SET [更新処理]; （例）全員のポイントを0にする UPDATE user SET point = 0; （結果） +------+--------+-------+ | id | name | point | +------+--------+-------+ | 1 | 山田 | 0 | | 2 | 鈴木 | 0 | | 3 | 加藤 | 0 | | 4 | 田中 | 0 | +------+--------+-------+ 4 「WHERE」で条件式をつける 上記3の書き方の場合、毎回更新がかかりポイントが0になってしまうので、条件を指定するWHERE句を使う。 4_1 基本的な書き方 UPDATE [テーブル名] SET [更新処理] WHERE [条件式]; （例）加藤さんだけ点数を150ポイントにする UPDATE user SET point = 150 WHERE name = "加藤"; （結果） +------+--------+-------+ | id | name | point | +------+--------+-------+ | 1 | 山田 | 0 | | 2 | 鈴木 | 0 | | 3 | 加藤 | 150 | | 4 | 田中 | 0 | +------+--------+-------+ Document
